## Getting Started

1. Fork the repository
2. Clone your fork locally
3. Create a new branch for your work:

   git checkout -b feature/your-feature-name

4. Install dependencies and run locally:

   npm install
   npm run dev

See `.env.example` for required environment variables.

## What to Avoid

- Committing secrets, tokens, or credentials
- Large refactors without discussion
- Adding new dependencies without clear justification
- Including personal data or scraped user content

---

## Code Style & Expectations

- Keep changes focused and readable
- Prefer small, purposeful commits
- Follow existing project structure and conventions
- Comment complex logic when helpful

---

## Submitting a Pull Request

1. Ensure your branch is up to date with `main`
2. Make sure the app builds and runs locally
3. Open a pull request with:
   - A clear description of what changed
   - Why the change is needed
